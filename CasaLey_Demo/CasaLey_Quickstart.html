2. Preparación del Entorno Snowflake

En esta sección habilitamos el ecosistema dedicado para Casa Ley con roles, almacenes virtuales y esquemas alineados a los lineamientos corporativos. Aprovechamos la separación de cómputo y almacenamiento de Snowflake para iniciar con un warehouse X-Small y escalar según la carga.

  1. Autentícate con SnowSQL y establece la base de trabajo: 
    snowsql -a <account> -u <usuario> -r ACCOUNTADMIN
  2. Ejecuta la hoja CasaLey_SnowSQL_Worksheet.sql ubicada en CasaLey_Demo: 
    !source /Users/gjimenez/Documents/GitHub/ipesa-pinturas/CasaLey_Demo/CasaLey_SnowSQL_Worksheet.sql
  3. Verifica que el warehouse CASALEY_WH esté disponible y en estado suspendido para iniciar bajo demanda.

Ejecuta el bloque clave para crear recursos dedicados:

create or replace role CASALEY_DEMO_ROLE;
create or replace warehouse CASALEY_WH
  warehouse_size = 'X-SMALL'
  auto_suspend = 60
  auto_resume = true;
create or replace database CASALEY_DB;
create or replace schema CASALEY_DB.CASALEY_ANALYTICS;
Ventaja Clave Cambia el tamaño del warehouse en segundos según la complejidad de las consultas, sin reprovisionar infraestructura ni interrumpir cargas.

3. Datos Sintéticos y Elasticidad

Generamos tablas transitorias con más de 15,000 filas sumadas entre clientes, ventas e inventario. Este volumen permite simular las cadenas de suministro y comportamiento de compra real en Casa Ley, mientras Snowflake automáticamente comprime y optimiza el almacenamiento.

Para validar la elasticidad:

  1. Escala el warehouse a SMALL con ALTER WAREHOUSE CASALEY_WH SET WAREHOUSE_SIZE = 'SMALL';
  2. Ejecuta la agregación de ventas por categoría incluida en la hoja de trabajo y observa la respuesta en segundos.
  3. Suspende el warehouse tras los cálculos para evitar consumo innecesario; Snowflake lo hará automáticamente tras 60 segundos de inactividad.

Genera los datos sintéticos con SnowSQL:

create or replace transient table CASALEY_DB.CASALEY_ANALYTICS.VENTAS_SINTETICAS as
select
  seq4() + 1 as venta_id,
  dateadd(day, - uniform(0, 120, random()), current_date()) as fecha_venta,
  ceil(uniform(1, 1000, random())) as tienda_id,
  round(uniform(50, 1500, random())::numeric(12,2), 2) as monto_ticket
from table(generator(rowcount => 12000));

Compresión Nativa

Los datos sintéticos se almacenan columnarmente, reduciendo el costo por registro sin configuraciones extra.

Auto-Suspend

Consumirás créditos solo mientras se ejecutan consultas; al terminar, el warehouse entra en pausa automáticamente.

4. Automatización y Gobierno

Snowflake simplifica la analítica continua con streams y tasks, además de políticas de masking administradas desde un solo punto. Esto asegura que Casa Ley pueda calcular ventas por hora en tiendas críticas y proteger segmentos de clientes sensibles.

Automatización en Minutos La tarea CASALEY_TASK_VENTAS_HORA procesa cada hora los cambios detectados por el stream y actualiza la tabla de métricas, sin servidores dedicados.
  • STREAM CASALEY_VENTAS_STREAM captura cambios incrementales sobre ventas.
  • TASK CASALEY_TASK_VENTAS_HORA opera con el mismo warehouse, asegurando consistencia y escalando bajo la misma política de costos.
  • La política de enmascaramiento CASALEY_SEGMENTO_MASK permite exponer datos agregados a socios sin revelar la clasificación de lealtad.
create or replace stream CASALEY_DB.CASALEY_ANALYTICS.CASALEY_VENTAS_STREAM
  on table CASALEY_DB.CASALEY_ANALYTICS.VENTAS_SINTETICAS;

create or replace task CASALEY_DB.CASALEY_ANALYTICS.CASALEY_TASK_VENTAS_HORA
  warehouse = CASALEY_WH
  schedule = '60 minute'
as
merge into CASALEY_DB.CASALEY_ANALYTICS.METRICAS_HORA tgt
using (
  select date_trunc(hour, fecha_venta) as ventana_inicio,
         categoria,
         sum(monto_ticket) as ventas_hora
  from CASALEY_DB.CASALEY_ANALYTICS.CASALEY_VENTAS_STREAM
  group by 1,2
) src
on tgt.ventana_inicio = src.ventana_inicio and tgt.categoria = src.categoria
when matched then update set ventas_hora = src.ventas_hora
when not matched then insert (ventana_inicio, ventana_fin, categoria, ventas_hora)
values (src.ventana_inicio, dateadd(hour,1,src.ventana_inicio), src.categoria, src.ventas_hora);

5. Colaboración Segura y Insights

Casa Ley puede compartir cuadros de mando y reportes segmentados con áreas de mercadotecnia, finanzas y operaciones gracias a vistas seguras y el modelo multi-tenant de Snowflake.

Vistas Seguras

VW_VENTAS_SEGMENTO permite compartir ventas por programa de lealtad sin exponer datos PI.

Data Sharing Controlado

Comparte sets agregados en segundos con filiales o socios de logística, sin mover datos fuera de Snowflake.

Integra estas vistas con herramientas de BI o dashboards de Casa Ley, aprovechando conectores nativos de Snowflake.

create or replace secure view CASALEY_DB.CASALEY_ANALYTICS.VW_VENTAS_SEGMENTO as
select
  v.fecha_venta,
  v.categoria,
  c.segmento_fidelidad,
  sum(v.monto_ticket) as ventas_segmento
from CASALEY_DB.CASALEY_ANALYTICS.VENTAS_SINTETICAS v
join CASALEY_DB.CASALEY_ANALYTICS.CLIENTES_DIM c on v.cliente_id = c.cliente_id
group by 1,2,3;

6. FinOps y Métricas de Costo

La disciplina FinOps es prioritaria para Casa Ley. Implementamos el monitor de recursos CASALEY_COST_MONITOR con umbrales de alerta al 80%, 90% y 100% del presupuesto diario.

  1. Configura el monitor al ejecutar la hoja SQL; queda ligado al warehouse CASALEY_WH.
  2. Consulta consumos recientes desde SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY usando el bloque provisto.
  3. Integra las métricas con dashboards financieros para decisiones de presupuesto diario en logística.
select
  usage_date,
  sum(credits_used) as creditos_consumidos,
  sum(credits_used) * current_account()::variant:"DATA_TRANSFER_CURRENCY_RATE" as costo_estimado
from snowflake.account_usage.warehouse_metering_history
where warehouse_name = 'CASALEY_WH'
  and usage_date >= dateadd(day, -7, current_date())
group by usage_date
order by usage_date;
Insight Accionable Con los créditos monitoreados diariamente, Casa Ley puede ajustar horarios de carga o suspender tareas no críticas al detectar desviaciones tempranas.
